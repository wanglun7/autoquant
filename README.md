# Binance 4h Research

Statistical market-structure research toolkit for Binance USDT-M perpetual futures.

## CLI workflow

The repository exposes exactly four commands, each focused on the statistical structure mainline.

```bash
binance4h fetch-data --data-dir data/raw --symbols BTCUSDT ETHUSDT        # download or refresh Binance 4h klines and funding tables
binance4h run-structure-scan --data-dir data/raw --output-dir results/structure_scan/scan_v1
binance4h run-structure-decompose --data-dir data/raw --output-dir results/structure_decompose/decompose_v1
binance4h run-structure-validate --data-dir data/raw --output-dir results/structure_validate/validate_v1
```

## Raw data layout

All commands expect the same `data/raw` foundation:

- `data/raw/klines/{symbol}_4h.csv`: each file contains Binance USDT-M 4h kline rows with ISO timestamps, open/high/low/close, volume, quote_volume, trade_count, taker_buy_base_volume, taker_buy_quote_volume, and close_time fields.
- `data/raw/funding/{symbol}.csv`: each file stores funding-rate events per symbol, with `fundingTime`, `fundingRate`, `markPrice`, and any metadata required by the structure scan helpers.

Files should be generated by `binance4h fetch-data` or seeded manually before running analysis.

## Structure outputs

Every structure command writes its artifacts under `results` so the mainline leaves a clear trace:

- `results/structure_scan/scan_v1/`: descriptive scan output and metadata for liquidity and imbalance profiles.
- `results/structure_decompose/decompose_v1/`: decomposed signal components derived from the scan matrices.
- `results/structure_validate/validate_v1/`: validation and sanity-check snapshots that prove the structure assumptions.

Use the same `--output-dir` flags when you need additional versions.

## Scope

- **Current boundary:** statistical market-structure research only. There are no backtests, academic replication routines, paper approximations, autoevolve loops, or trading autoresearch systems in this worktree.
- **Future direction:** time-series research will be added later, building on the same `data/raw` foundation so the structure outputs remain reusable for forecasting efforts.
